# exec-plans/ — 執行計劃

每個任務一份計劃，記錄為什麼做、做什麼、留下什麼成果。檔名用英文 kebab-case（如 `phase3-docker-deploy.md`）。

- `active/` — 進行中的任務。
- `completed/` — 已完成的任務。

## 模板

新任務從這份骨架開始，建檔於 `active/`：

```
# <任務標題>
- 狀態：active
- 期間：YYYY-MM-DD →
- 證明：—

## 背景
任務的起點與動機：目前的問題、為什麼要做。

## 計劃
逐項列出要做的事，每項附上當時做此決策的理由。

## 驗證
本任務加上的 tests（實作的 invariant）；無則「無」。

## 成果
成品展示：截圖、錄影、demo 連結等。
```

## 完成

任務完成時，把 `狀態` 改為 `completed`、補上 `期間` 完成日與 `證明`（PR / commit），填寫 `驗證` 與 `成果`，再把檔案移到 `completed/`。

`背景` 與 `計劃` 在開任務時寫定後不再更動，記錄的是當時的意圖與理由。
