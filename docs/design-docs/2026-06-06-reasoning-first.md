# 先推理再輸出

## 介紹
LLM 結構化輸出的 schema，`reasoning` 欄要排在所有答案／內容欄位前面。純粹的輸入欄位（從使用者輸入直接取出、給推理當依據）例外，可以更前面——重點是 reasoning 在答案之前，不一定是第一欄。

## 為什麼
結構化輸出是照欄位順序生成的；把 reasoning 放前面，等於把思考過程寫進 schema，模型會先推理再作答，後面的欄位都根據這段推理生成，品質比較穩定。之後新增任何生成型 schema，預設都把 reasoning 排在答案欄前面。
