# design-docs/ — 決策紀錄

每個檔記錄一個設計決定——換個專案可能會做不一樣的選擇（例如伺服端要不要幫使用者存憑證）。檔名用 `YYYY-MM-DD-英文 kebab`（如 `2026-06-06-reasoning-first.md`），日期是做決定那天，內容用中文。決定改了就改原檔、在「變更紀錄」記一筆，不另開新檔。

## 模板
```
# <決策標題>

## 介紹
這個決定是什麼：一句講清楚、照著做得到的主張。

## 為什麼
為什麼這樣決定，以及之後遇到類似情況怎麼判斷。

## 相關測試
把關這個決定的測試；還沒有就填「無」。

## 變更紀錄
哪天改了什麼、為什麼；沒改過就填「無」。
```
